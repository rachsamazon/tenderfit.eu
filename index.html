How it works

  1. Parse: reads each tender folder (XML/PDF/meta) and builds a compact “tender card”.
  2. Sieve: cheap rules (CPV + keywords + language + deliverable type) reject most tenders instantly.
  3. Judge: LLM reviews only survivors and returns strict JSON: label, why, English summary, red flags.
  4. Organize: writes summary_en.txt + llm_verdict.json, optionally renames folders.

What TenderFit considers “high leverage”

  • Digital-only deliverables: software, web platform, SaaS, cybersecurity, data/AI, documentation, training.
  • Low paperwork, small-scope, remote-friendly. If unsure between Y and R, it chooses R.
  • If tender language is not English or Croatian → auto R.
  • If country ≠ Croatia and permanent physical presence is required → downgrade (or reject).
